Working with Roster Simulator lines
Note
Roster Simulator lines are only saved during the session. Once you logout or exit the system, the Roster Simulator line you built is no longer available. Any bids created from Roster Simulator lines will not be removed.
Showing Roster Simulator line
To show the Roster Simulator line:
•
In the Trip explorer menu, select the Show Roster Simulator line command.
The Roster Simulator line is displayed in the Calendar, including any trips that were originally placed on the line in the session.
Hiding Roster Simulator line
To hide the Roster Simulator line:
•
In the Trip explorer menu, select the Hide Roster Simulator line command.
The line is collapsed from the display and any trips placed on the line remain there during the session.
Adding trips to Roster Simulator line
To add trips to the Roster Simulator line:
•
In the Trip work area, select one or more trips and select the Tools> Add Trips command. Alternatively, in the Gantt chart, you can add trips to the line by double-clicking them.
The trips are added to the Roster Simulator line and the added trips are highlighted in the Trip work area.
Removing trips from Roster Simulator line
To remove trips from the Roster Simulator line:
•
Select one or more trips in the Roster Simulator line and click Remove from line in the Calendar.
The trips are cleared from the Roster Simulator line.
Clearing Roster Simulator line
As only one Roster Simulator line can be built at a time, you must clear the trips from the existing line before you can start building a new line. To clear the Roster Simulator line entirely:
•
In the Calendar, click Clear.
All trips are removed from the Roster Simulator line.
Performing legality check
You perform a legality check before sending the bids to the optimizer, in case there are errors or basic rules that can be checked by the system.
To send the Roster Simulator line to the rostering system for a legality check:
•
Click Legality check.
The line information is updated, based on values returned from the Rostering System. Illegalities, if any, are displayed in the Validation view.
