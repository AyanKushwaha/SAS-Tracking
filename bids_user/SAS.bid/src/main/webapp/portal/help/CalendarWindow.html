Calendar
The calendar, located in the top of the application, displays days, weeks and months and additional information such as week numbers and weekends.
You can quickly navigate to a specific date with the Jump-to-date calendar.
In the calendar, pre-assignments are displayed on the Pre-assignment line. In the calendar you can activate the Roster Simulator line for entering trips manually from the Trip work area. On a separate line, you can display the Bid Impact overview containing trip bids for the bid period. When you select Request manager, the Available Days Off overview is displayed on a separate line.
Bid period
In the calendar you can see the period being planned, called the bid period. The current bid period can be either open or closed (not yet open).
Dimmed calendar elements are not included in the current bid period and not available for bidding.
In the Bid explorer, the Bid window is displayed, this is the period during which you can create and submit bids for the bid period.
When you select Request manager, the calendar displays the request period for the selected request category.
Navigation
To navigate in the calendar click the backward arrow or forward arrow . Dependent on the installation, you can view actual (completed) schedules from previous bid periods displayed on the Pre-assignment line by navigating back in time.
You can create bids from the calendar by selecting a day, week or period, see Creating calendar-based bids.
Jump-to-date
To quickly navigate to a specific date in the calendar, click the Jump-to-date icon . A pop-up calendar is displayed, with the first date in the bid period in the calendar selected. To navigate to another date, select a date in the pop-up calendar or click the Today button to select today’s date. The calendar timeline is updated, starting with the selected date.
Pre-assignment line
Pre-assignments are shown on a timeline, directly under the Calendar timeline, displaying pre-assignments and completed schedules for previous bid periods. This line is always visible.

Buttons
The Roster Simulator line has the following buttons:
Clear
Clears the Roster Simulator line, see Clearing Roster Simulator line.
Remove from line
Removes selected trip from the Roster Simulator line, see Removing trips from Roster Simulator line.
Legality check
Checks Roster Simulator line for validation errors, see Performing legality check. Errors are displayed in the Validation view.
Located in the area below the Roster Simulator line, a number of line information values (as for example total block, total credit) are displayed that:
•
are calculated upon receiving the results of a legality check
•
are hidden until the first time legality check is completed
•
are invalidated if you modify the line, for example by adding/removing trips.

Bid Impact overview
 
The Bid Impact overview is located under the Pre-assignment line in the calendar. The Bid Impact overview contains two rows on a timeline:
•
The top row contains trips matching bids from the selected bid scenario.
•
The bottom row contains non-working bids, for example day-off bids. Only bids with a validity period of 7 days or less are shown.
Both rows are updated as you create, modify or remove bids.
•
To view a count of the number of trips touching a particular date, hover over a colored cell in the top row.
•
To view detailed information on each bid, hover a colored cell in the bottom row.
•
To show trips in a list, click , or a colored bar on the top row. The Trip window is displayed, showing trip details. You can directly create bids matching a specific pattern of trips or blocks, see Creating Specific Pattern bids.

Available Days Off
The Available Days Off overview is directly located under the Pre-assignment line in the Calendar. It shows the available slots/days off per day in the calendar.
Note
The Available Days Off overview is only available when you select Request manager.
