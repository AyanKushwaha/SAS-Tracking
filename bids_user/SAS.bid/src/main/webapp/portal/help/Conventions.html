Type styles
This documentation uses the following conventions for type styles:
Italic
Cross-references.
Bold
Names of GUI components, such as buttons, boxes, fields, menus and commands.
Courier
Data and choices entered such as file names, values and parameters.
Initial Capitals
Names of forms and services, for example Plan Manager.
>
Menu choices.
For example Hard Locks> Set means select Hard Locks from the menu and select Set from the submenu.
BNF notation
This documentation uses BNF notation when describing code syntax. The following meta symbols are used:
::=
Is defined as.
|
or
*
The preceding item is repeated zero or more times.
+
An item is repeated one or more times.
(x1 | x2)*
Parentheses are used to group items.
[remark]
Optional items are enclosed in meta symbols (square brackets).
The following type styles are used:
variable
Distinguishes syntax rule names (also called
non-terminal symbols or category names).
parameter
Indicates terminal symbols (reserved words), which are to be written exactly as presented.
